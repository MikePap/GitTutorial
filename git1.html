Configurazioni

Identità

git config --global user.name "Jon"
git config --global user.email "jon@gmail.com"

Con l'opzione --global dovrai farlo solo una volta, dopo di che Git userà sempre queste informazioni per qualsiasi operazione fatta sul sistema

Editor predefinito

git config --global core.editor gedit

Diff (è lo strumento diff predefinito, da usare per risolvere i conflitti di merge)

git config --global merge.tool vimdiff

Controllo impostazioni

git config --list				(elenca tutte le impostazioni)

Per controllare una in specifico, es:

git config user.name

Esclusione file

git config --global core.excludesfile <nomeFile>

E' un'alternativa al file .gitignore (vedere cap.2: Ignorare file). Nel file passato come argomento (completo di percorso) si possono inserire delle regole per escludere alcuni tipi di file dal tracciamento. Questa configurazione è esterna al progetto (a differenza di .gitignore) e quindi una volta configurata è valida per tutti i futuri progetti. Ciò non toglie che .gitignore possa sovrascrive il file di questa configurazione.

Personalizzare l'output coi colori

git config --global color.ui true

Attiva tutti i colori di default nel terminale. In questo modo Git visualizzerà i suoi messaggi con colori vari in modo da renderli più leggibili.
Nel caso in cui si desideri una configurazione più specifica per comando, o si abbia una versione meno recente, Git fornisce impostazioni di colorazione verb-specific. Ognuna di esse può essere impostata a true, false oppure always: 

color.branch
color.diff
color.interactive
color.status

Configurare l'esportazione

Alcune impostazioni possono anche essere specificate per un percorso, per fare in modo che Git applichi queste impostazioni solo per una sottodirectory o un sottoinsieme di files.
Queste opzioni path-specific sono chiamate attributi di Git e sono impostati nel file .gitattributes in una delle directories del progetto (normalmente nella radice del progetto) oppure nel file .git/info/attributes se non si vuole che venga fatto il commit del file degli attributi del proprio progetto.

Usando gli attributi, si possono fare cose come specificare strategie di merge separate per files individuali o directories nel proprio progetto, comunicare a Git come fare diff per files non di testo, oppure fare in modo che Git filtri il contenuto prima del check all'interno o all'esterno di Git.
Gli attributi Git permettono inoltre di attuare operazioni interessanti quando si esporta un archivio del proprio progetto.
È possibile comunicare a Git di non esportare alcuni files o directories quando si genera un archivio.
Nel caso ci sia una sottodirectory o file che non si vuole includere nell'archivio ma che si vuole comunque tenere nel progetto, è possibile determinare questi files tramite l'attributo export-ignore.

Per esempio, diciamo di avere dei files per i test in una sottodirectory test/, non avrebbe senso includerla in un tarball del progetto. È possibile aggiungere la seguente riga al file Git attributes: 

test/ export-ignore

Ora, quando verrà eseguito git archive per creare la tarball del progetto, la directory non sarà inclusa nell'archivio.

Ottenere Aiuto

Ci sono tre modi per vedere le pagine del manuale (manpage) per ogni comando di Git:

git help 
git <verb> --help
man git-<verb>

Per esempio, per consultare la pagina del manuale per il comando config digitanto 

git help config

Il progetto di Git fornisce un documento che da molti suggerimenti circa le commit per/da cui creare patch: puoi trovarlo nel codice sorgente di Git nel file Documentation/SubmittingPatches'
